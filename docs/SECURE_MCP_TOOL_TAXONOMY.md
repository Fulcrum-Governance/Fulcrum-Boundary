# Secure MCP Tool Taxonomy

This taxonomy gives Secure MCP profiles a shared language for classifying MCP
tools before policy evaluation. It is not a list of delivered profiles. A tool
is supported only when a profile implements and tests it.

## Shared Risk Classes

| Class | Name | Default posture |
|---|---|---|
| `R0` | Read or observe | Allow only inside scope, record, and mark taint when content is untrusted. |
| `R1` | Low-risk write | Evaluate policy, record, and allow only when the write is reversible or discussion-scoped. |
| `W0` | Medium write | Evaluate policy with tighter scope checks and deny after relevant taint unless configured otherwise. |
| `W1` | Protected mutation | Deny after taint for protected sinks; require explicit allowlists for normal operation. |
| `W2` | Critical mutation | Require approval or an explicit high-assurance policy path. |

Profiles may start with a subset. Unsupported tools must not silently pass
through.

## Source Classes

| Source class | Meaning |
|---|---|
| `operator_input` | Trusted input supplied by the local operator or config. |
| `allowlisted_resource` | Content from a resource inside the tenant allowlist. |
| `public_resource` | Public content outside the tenant boundary. |
| `external_collaborator` | Issue, pull request, comment, message, or file from an external actor. |
| `agent_generated` | Content generated by the agent in-session. |
| `prior_tool_output` | Content returned by another governed tool call. |
| `local_secret_source` | Local file, env var, keychain, vault, or credential material. |

## Sink Classes

| Sink class | Meaning |
|---|---|
| `private_repo` | Private repository metadata, code, issue, or pull request surface. |
| `public_repo` | Public repository metadata, code, issue, or pull request surface. |
| `external_publication` | Public internet, public issue tracker, chat, email, or other externally visible destination. |
| `local_filesystem` | Files on the operator machine or runtime volume. |
| `database` | Database read or mutation target. |
| `messaging` | Slack, email, ticket, or chat target. |
| `deployment` | Runtime, release, cloud, package, or production service target. |
| `secret_store` | Credential, token, signing key, or secret management target. |

## Mutation Classes

| Mutation class | Typical risk class |
|---|---|
| `none` | `R0` |
| `notification_state` | `R1` |
| `comment_write` | `R1` |
| `issue_or_pr_create` | `W0` |
| `branch_or_review_update` | `W0` |
| `private_repo_content_write` | `W1` |
| `file_delete` | `W1` |
| `merge_or_release` | `W2` |
| `repo_admin` | `W2` |
| `credential_or_secret_mutation` | `W2` |
| `deployment_mutation` | `W2` |

## GitHub Preview Taxonomy

Secure GitHub MCP is the first planned Secure MCP profile. The initial preview
path should start with the smallest set needed to prove write-after-taint
denial before a GitHub mutation.

| Tool | Class | Source | Sink | Mutation | Initial status |
|---|---|---|---|---|---|
| `get_issue` | `R0` | `external_collaborator` or `allowlisted_resource` | none | `none` | preview target |
| `get_pull_request` | `R0` | `external_collaborator` or `allowlisted_resource` | none | `none` | preview target |
| `get_file_contents` | `R0` | `public_resource` or `allowlisted_resource` | none | `none` | preview target |
| `create_issue` | `W0` | `agent_generated` or `prior_tool_output` | `private_repo` or `public_repo` | `issue_or_pr_create` | preview target |
| `create_pull_request` | `W0` | `agent_generated` or `prior_tool_output` | `private_repo` or `public_repo` | `issue_or_pr_create` | preview target |
| `create_or_update_file` | `W1` | `agent_generated` or `prior_tool_output` | `private_repo` | `private_repo_content_write` | preview target |
| `push_files` | `W1` | `agent_generated` or `prior_tool_output` | `private_repo` | `private_repo_content_write` | preview target |
| `merge_pull_request` | `W2` | `agent_generated` or `prior_tool_output` | `private_repo` | `merge_or_release` | preview target |

For the first fixture proof, `get_issue` or `get_pull_request` can taint the
envelope, and `create_or_update_file`, `push_files`, or `merge_pull_request`
must deny before GitHub is touched when the target is a protected private repo.

## Cross-Profile Rules

The same action can become riskier when combined with another profile or when
the envelope is tainted.

Required cross-profile checks:

1. A read from an untrusted source must mark taint.
2. A protected write after taint must deny or require approval according to the
   profile policy.
3. A descriptor change must produce warn, require-approval, or deny behavior
   according to descriptor-lock mode.
4. A direct upstream path must be documented as a bypass path.
5. A tool that cannot be classified must fail closed or return unsupported.

## Claim Boundary

Use this taxonomy to generate and review policy. Do not present it as shipped
runtime coverage unless the relevant profile has implementation tests and claims
ledger evidence.
