# Fulcrum Boundary v0.10.0

Fulcrum Boundary v0.10.0 is the distribution release: the first release with
prebuilt binaries, a one-command Homebrew install, and decision records whose
canonical bytes follow RFC 8785 (JCS) — so an independent, stock RFC 8785
implementation recomputes the same `decision_hash` a Boundary build produced.

The release story stays narrow. Boundary decides before execution for routed
agent-tool actions, records the verdict in a hash-verifiable decision record,
and now ships through a tag-gated pipeline that anyone can install in one
command and verify with stock tooling. It does not add a new governed action
surface, does not add a transport adapter, and does not upgrade any preview
surface to production.

MCP remains the first production route. Secure GitHub, Command Boundary, and
Edit Boundary remain preview surfaces.

## Install

First release with prebuilt binaries. The Homebrew formula and the release
archives install the static (`CGO_ENABLED=0`) build:

```bash
brew install fulcrum-governance/tap/boundary
boundary selftest
boundary demo github-lethal-trifecta    # Lane 1: MCP, the first production route
boundary demo tamper-evidence           # forge the receipt, watch verification refuse it
```

In the static build the Postgres AST classifier is unavailable: every routed
SQL statement classifies as `UNKNOWN` and is denied fail-closed, so the static
build never allows SQL the cgo build would deny. The `_cgo` archives carry the
full classifier. Building from source requires Go 1.25+ and, for the default
cgo build, a C toolchain:

```bash
go install github.com/fulcrum-governance/fulcrum-boundary/cmd/boundary@v0.10.0
```

The default first-run path uses fixture data. It makes no live calls and
performs no real mutations.

## What Changed

- **Prebuilt distribution.** A tag-gated release pipeline builds static
  archives for macOS and Linux with checksums, publishes a container image,
  and pushes the Homebrew formula to `fulcrum-governance/homebrew-tap`.
  Releases up to and including v0.9.0 shipped source-only.
- **RFC 8785 (JCS) canonical decision records.** The canonical preimage behind
  every stable hash — `decision_hash`, `request_hash`, the raw request-hash
  counterpart, and `policy_bundle_hash` — now follows RFC 8785. This is a
  pre-1.0 format change: records emitted by older builds no longer verify
  under this build. The conformance statement is scoped to the decision
  record; it is not a claim that Boundary as a whole is standards-conformant.
- **A standalone Python verifier** under `verifiers/python/` recomputes
  `decision_hash` with the stock `rfc8785` package — no Boundary code in the
  loop. Committed conformance vectors pin the Go and Python implementations to
  the same canonical bytes, enforced in CI.
- **`boundary demo tamper-evidence`** — the forge-the-receipt loop as a guided,
  fixture-only demo: mutate a recorded verdict, watch `verify-record` refuse
  it.
- **First-run and CLI polish.** `--version`/`-v`, `boundary help <topic>`,
  rich help across the bare commands, `--json` on `verify`
  (`boundary.verify.v1`) and `verify-record` (`boundary.verify_record.v1`),
  TTY color in demos, and first-run environment diagnostics in
  `boundary doctor`.
- **A Claude Code `PreToolUse` hook** routes hook-delivered tool calls through
  `boundary command classify` before execution. Routed-only: the hook governs
  only the calls Claude Code sends through it.
- **Hardening.** Bounded inbound HTTP request bodies and server timeouts on
  the gateway; policy-evaluator faults surfaced with opt-in fail-closed
  routing per transport; malformed static-policy match blocks fail closed;
  toolchain floored to `go1.26.4` for the patched standard library.
- **Honesty surface.** The public-language lint now scans the docs site
  recursively plus the Command Boundary, Edit Boundary, and release-notes
  trees; a skeptic's FAQ, a category comparison, a standards and incident
  mapping, and a complete CLI reference command map shipped with it.

## Verify a Record Without Boundary

The record-trust loop now closes with tooling Boundary does not own:

```bash
boundary demo github-lethal-trifecta --out lane1.txt
boundary verify-record github-lethal-trifecta-artifacts/decision-record.json

pip install rfc8785
python3 verifiers/python/boundary_verify.py \
  github-lethal-trifecta-artifacts/decision-record.json
```

Both recompute the same `decision_hash` from the record bytes. The hashes are
unkeyed SHA-256 over RFC 8785 canonical bytes: integrity, not authenticity. A
passing check detects post-emission tampering; it does not prove who produced
the record, that the action was executed or prevented, or that the verdict was
correct.

## Two Proof Lanes

| Lane | Status | Demo | What is denied | Verified shape |
| --- | --- | --- | --- | --- |
| MCP — the first production route | production | `boundary demo github-lethal-trifecta` | A write-after-taint GitHub action, denied before upstream | `actual=DENY`, `upstream_called=false`, `reason=lethal_trifecta_detected` |
| Command Boundary — a delivered preview (routed-only) | delivered preview | `boundary demo command-secret-exfil` | A routed `curl -d @.env ...` secret exfiltration, denied before execution | `actual=DENY`, `executed=false`, `class=C6` |

## What It Does Not Prove

| Limit | Why it matters |
| --- | --- |
| Universal attack prevention | Fixture demos and policy tests cover supplied paths only. |
| Production Secure GitHub | Secure GitHub remains preview until deployment bypass evidence and broader live coverage exist. |
| Production Command Boundary | Command Boundary remains routed-path-only preview; direct shell access is not governed. |
| Production Edit Boundary | Edit Boundary remains routed-edit-envelope-only preview. |
| Deployment bypass resistance | A passing demo or policy test does not prove every direct or unrouted path to the same tool has been removed. |
| Blanket standards conformance | The RFC 8785 statement is scoped to the decision record's canonical bytes; it is not a whole-product conformance claim. |
| Cryptographic record authenticity | The record hashes prove integrity, not authenticity; the hashes are unkeyed and Boundary does not emit `proved` decisions. |
| Static-build SQL classification | The static build classifies routed SQL as `UNKNOWN` and denies it fail-closed; it does not classify statements the cgo build would. |
| Hook coverage | The Claude Code hook governs only hook-delivered tool calls; it does not govern direct shell or editor access outside the hook. |
| Topology attestation | `topology_profile` records an asserted posture; it does not verify the deployment matches it. |
| Independent no-upstream proof | `execution_claim` is an adapter self-report; `boundary replay` reproduces the decision, not the absence of upstream side effects. |
