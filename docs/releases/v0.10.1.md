# Fulcrum Boundary v0.10.1

Fulcrum Boundary v0.10.1 is the first release published with prebuilt
binaries. It is content-identical to [v0.10.0](./v0.10.0.md) plus the release
pipeline fix that lets the publish step run; the full feature story for the
v0.10.x line — RFC 8785 (JCS) canonical decision records, the standalone
Python verifier, the tamper-evidence demo, the Claude Code hook, static
builds, and the CLI polish — is documented in the
[v0.10.0 release notes](./v0.10.0.md).

What v0.10.1 changes over v0.10.0:

- The release pipeline's evidence-bundle staging directory
  (`.release-extras/`) is gitignored, so goreleaser's dirty-state check
  passes and the publish step runs.
- The pipeline fails fast with a clear error when the Homebrew tap token
  secret is empty, instead of failing after the full build.

v0.10.0 was tagged but its pipeline run failed before publishing any assets,
so that tag has no binaries, formula, or container image; it remains a valid
source-install target.

## Install

```bash
brew install fulcrum-governance/tap/boundary
boundary selftest
boundary demo github-lethal-trifecta    # Lane 1: MCP, the first production route
boundary demo tamper-evidence           # forge the receipt, watch verification refuse it
```

The formula and the release archives install the static (`CGO_ENABLED=0`)
build: the Postgres AST classifier is unavailable, every routed SQL statement
classifies as `UNKNOWN` and is denied fail-closed, and the static build never
allows SQL the cgo build would deny. The `_cgo` archives carry the full
classifier. Building from source requires Go 1.25+ and, for the default cgo
build, a C toolchain:

```bash
go install github.com/fulcrum-governance/fulcrum-boundary/cmd/boundary@v0.10.1
```

The default first-run path uses fixture data. It makes no live calls and
performs no real mutations.

## What It Does Not Prove

The [v0.10.0 limits table](./v0.10.0.md#what-it-does-not-prove) applies to
v0.10.1 unchanged: fixture demos cover supplied paths only; record hashes
prove integrity, not authenticity; the RFC 8785 statement is scoped to the
decision record and is not a whole-product conformance claim; preview
surfaces remain preview, and no surface changed maturity in v0.10.1.

## Verification

The release package is validated with:

```bash
make release-check
make docs-build
go test ./claims/... -count=1
go test ./... -count=1 -timeout 5m
go vet ./...
git ls-files '*.go' | xargs gofmt -l
```
