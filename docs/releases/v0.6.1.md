# Fulcrum Boundary v0.6.1

Fulcrum Boundary v0.6.1 packages the v0.6.x utility train. It makes the current
Boundary release easier to prove, diagnose, and hand off without adding a new
governed action surface.

MCP remains production. Secure GitHub, Command Boundary, and Edit Boundary
remain preview surfaces.

## Install

Requires Go 1.25+.

```bash
go install github.com/fulcrum-governance/fulcrum-boundary/cmd/boundary@v0.6.1
boundary selftest
boundary demo github-lethal-trifecta
```

The default first-run path uses fixture data. It makes no live GitHub calls and
performs no real mutations.

## What Shipped

- `boundary version` for local build metadata in text and JSON form.
- `boundary demo action-boundary` for a fixture-only cross-surface demo across
  MCP / Secure GitHub, Command Boundary, and Edit Boundary.
- `boundary doctor` local diagnostics for MCP, Command Boundary, and Edit
  Boundary routed-surface readiness and bypass caveats.
- `boundary evidence bundle` and `boundary evidence verify` for local evidence
  manifests, SHA-256 hashes, fixture-safe utility outputs, and bundle integrity
  checks.
- `make release-check` coverage for the utility commands above.
- Active install and GitHub Action examples aligned to `@v0.6.1`.

## Utility Path

```bash
boundary version
boundary doctor --json
boundary demo action-boundary
boundary evidence bundle --include-demo --out boundary-evidence
boundary evidence verify boundary-evidence
```

## What It Proves

| Surface | Proof |
| --- | --- |
| Version | The binary can report local build metadata, Go runtime, module path, and schema-versioned JSON. |
| Action Boundary demo | Fixture MCP / Secure GitHub, Command Boundary, and Edit Boundary examples can be shown together without live mutation. |
| Doctor | Local diagnostics can report routed-surface readiness and bypass caveats without network calls. |
| Evidence bundle | Local release evidence can be packaged with a manifest and SHA-256 artifact hashes. |
| Evidence verify | Bundle integrity can be checked locally before handoff or release review. |
| Release check | The repeatable release gate exercises the utility train in addition to existing tests and demos. |

## What It Does Not Prove

| Limit | Why it matters |
| --- | --- |
| Production route enforcement | Utility evidence is local proof packaging, not deployment topology proof. |
| Universal attack prevention | Fixture demos cover selected paths only. |
| Production Secure GitHub | Secure GitHub remains preview until deployment bypass evidence and broader live coverage exist. |
| Production Command Boundary | Command Boundary remains routed-path-only preview. |
| Production Edit Boundary | Edit Boundary remains routed-edit-envelope-only preview. |
| Cryptographic release provenance | `boundary version` reports metadata; it is not a signing or attestation system. |

## Maturity Table

## Claims Boundary

Use:

> Boundary v0.6.1 adds local utility commands for version reporting, fixture
> action-boundary demos, routed-surface diagnostics, and evidence bundle
> verification.

Do not claim:

- Evidence bundles prove production safety.
- Doctor proves all routes are protected.
- The action-boundary demo proves all attacks are blocked.
- Version output proves cryptographic release provenance.
- Any preview surface is production.
