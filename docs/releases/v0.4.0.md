# Fulcrum Boundary v0.4.0

Date: 2026-05-27

Fulcrum Boundary v0.4.0 packages the Command Boundary preview as the next
release after the v0.3.0 MCP Firewall and Secure GitHub release.

## Install

Requires Go 1.25+.

```bash
go install github.com/fulcrum-governance/fulcrum-boundary/cmd/boundary@v0.4.0
boundary selftest
boundary demo github-lethal-trifecta
```

The original v0.3.0 release story remains intact: MCP is still the production
adapter, Secure GitHub remains preview, and the GitHub lethal-trifecta demo uses
fixture data only.

## Command Boundary Preview

Command Boundary adds project-local command governance for commands routed
through Boundary:

```bash
boundary command classify -- git push origin main
boundary command run -- git status
boundary shell
```

## What Shipped

- `boundary command classify` classifies command risk without execution.
- `boundary command run` evaluates wrapper-routed commands before execution.
- Command decision records capture command class, action, execution status,
  reason, cwd, argv hash, and redacted args.
- `boundary command install --project` creates reversible project-local shims in
  `.boundary/bin`.
- `boundary command uninstall --project` removes Boundary-generated project
  shims.
- `boundary shell` launches a scoped subshell with project shims first on
  `PATH`.
- Command redteam packs exercise fixture-only cleanup, secret-exfiltration, and
  repo-mutation command-risk paths without live mutation.

## What It Proves

- Boundary can classify command risk without executing commands.
- Boundary can evaluate wrapper-routed commands before execution.
- Denied or approval-required commands do not execute.
- Allowed commands execute through explicit argv forwarding, not shell
  interpolation.
- Project-local shims can route selected commands through Boundary when the
  operator opts in.
- Fixture command redteam packs report command-risk outcomes without live
  mutation.

## What It Does Not Prove

- Global shell control.
- Protection for direct shell access.
- CI, SSH, cron, editor, or arbitrary-process control by default.
- Shell sandboxing.
- Production command governance.
- Governance of direct file edits outside routed command paths.
- Universal coding-agent safety.

## Maturity

| Surface | Status | Release truth |
| --- | --- | --- |
| MCP | production | Unchanged from v0.3.0. |
| Secure GitHub | preview | Unchanged from v0.3.0; fixture-backed until live conformance evidence exists. |
| Command Boundary | preview | Delivered for routed command paths only. |

## Safe Copy

Boundary provides preview project-local command governance for commands routed
through `boundary command run`, `boundary shell`, or project-local shims.

## Forbidden Copy

Do not claim that Boundary controls all shell commands, protects direct shell
access, prevents every overeager agent action, provides production command
governance, provides shell sandboxing, or governs direct file edits outside
routed command paths.
